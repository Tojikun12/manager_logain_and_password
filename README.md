# 🔐 Password Organizer – Менеджер паролей (консольный)

Консольное приложение на C++ для безопасного хранения логинов и паролей. Данные шифруются двумя методами (Цезарь или XOR) и сохраняются в бинарные файлы. Доступ к программе защищён мастер-паролем.

---

## 📌 Возможности

- 🔒 **Фиксированный мастер-пароль** (по умолчанию `jjiolk`) – доступ к данным только после его ввода.
- 🗂 **CRUD операции** с записями: добавление, просмотр всех, просмотр по ID, редактирование, удаление.
- 🔐 **Два алгоритма шифрования**:
  - Шифр Цезаря (сдвиг вычисляется на основе мастер-пароля)
  - XOR-шифрование (мастер-пароль используется как ключ)
- 📁 **Раздельное хранение** для каждого метода:  
  - `passwords_caesar.dat` – для Цезаря  
  - `passwords_xor.dat` – для XOR
- 🎲 **Генерация надёжных паролей** (заданной длины, с проверкой сложности)
- 📤 **Экспорт всех записей в текстовый файл** (формат TXT с разделителем табуляции)
- 📥 **Импорт записей из текстового файла** (с автоматической проверкой дубликатов)
- 🛡 Проверка надёжности пароля (длина ≥ 8, наличие букв разного регистра, цифр, спецсимволов)

---

## 🛠 Технологии

- **Язык**: C++17
- **Стандартная библиотека**: STL (`iostream`, `fstream`, `vector`, `string`, `random`, `algorithm`, `cctype`)
- **Кодировка**: UTF-8 (для корректного отображения русских символов)
- **Сборка**: компилятор с поддержкой C++17 (MinGW, MSVC, Clang)

## 🚀 Использование

### Вход в программу
При запуске программа запрашивает мастер-пароль. По умолчанию он установлен как **`jjiolk`**.  
После успешного ввода вы попадёте в главное меню.

### Главное меню
```
ГЛАВНОЕ МЕНЮ
1. Добавить запись
2. Показать все
3. Показать по ID
4. Редактировать
5. Удалить
6. Экспорт в TXT
7. Импорт из TXT
8. Выйти
```

### Добавление записи
- Введите название сервиса и логин.
- Программа проверит уникальность логина в рамках этого сервиса.
- Выберите способ ввода пароля:
  - Вручную (с проверкой надёжности)
  - Автоматическая генерация (укажите длину от 8 до 25)
- Запись шифруется и сохраняется.

### Просмотр всех записей
Отображает ID, название сервиса и логин (пароль не показывается для безопасности).

### Просмотр по ID
Показывает полную информацию, включая пароль (расшифровывается на лету).

### Редактирование и удаление
Работают по ID записи.

### Экспорт в TXT
Создаёт текстовый файл (по умолчанию `passwords_export.txt`) со всеми записями в расшифрованном виде.  
Формат: `ID\tСервис\tЛогин\tПароль`.

### Импорт из TXT
Считывает записи из текстового файла, шифрует их и добавляет в базу. Пропускает дубликаты (по паре сервис+логин).

## 🗂 Файловая структура

В рабочей директории создаются два файла данных (в зависимости от выбранного метода):

- `passwords_caesar.dat` – бинарный файл для шифра Цезаря.
- `passwords_xor.dat` – бинарный файл для XOR.

Каждый файл содержит:
- 1 байт – метод шифрования (1 – Цезарь, 2 – XOR)
- 8 байт – количество записей (size_t)
- Для каждой записи: ID, длина и содержимое трёх строк (сервис, логин, пароль)

---

## 🔐 Механизм шифрования

### Шифр Цезаря
- Вычисляется сдвиг: сумма ASCII-кодов символов мастер-пароля приводится к диапазону 3–28.
- Каждая буква сдвигается на это значение (по модулю 26).
- Цифры сдвигаются по модулю 10.
- Остальные символы остаются без изменений.

### XOR-шифрование
- Каждый байт данных XOR-ится с соответствующим байтом мастер-пароля (циклически).
- Благодаря симметричности XOR дешифрование выполняется той же операцией.

## 💡 Дополнительные замечания

- Для смены мастер-пароля необходимо изменить значение в исходном коде (переменная `masterPassword` и условие проверки `if (entered != "jjiolk")`).
- Программа рассчитана на работу в Windows (используется `SetConsoleCP` и `SetConsoleOutputCP` для корректной кодировки). При переносе на Linux/Unix эти строки можно закомментировать.
- Все операции с файлами производятся в той же папке, где находится исполняемый файл.
